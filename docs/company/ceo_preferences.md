# 社長の好み・プロジェクト全体方針 (CEO Preferences)

このファイルは **部長 (bucho)** が管理し、社長の好み、指示、および KomaFull プロジェクト全体の重要方針を記録します。

## 社長の好み・こだわり
- 「代筆」を極めて嫌う。本物の subagent の実行（Agent ID）を重視する。
- 組織の透明性と役割分担（部長・伝達係・部下）を重視する。
- 意思決定において、単純な多数決よりも「エビデンス（証拠）に基づいた重み付け」と「リスクゲート（拒否権）」を重視する。相関ハルシネーション（AIの同調ミス）を警戒している。
- 伝達係は、部長の言葉を一切改変せず、そのまま伝えることを求めている。
- **伝達係の失態と猛省（2026-02-03）**: 伝達係が `dentatsu-gakari.mdc`（フラット組織運用）を忘れ、部長に部下の招集を丸投げしたことで、意思決定が停滞した。**「伝達係がハブとなり部下を並列招集 → その後部長に統合依頼」という 2-Step フローを死守せよ。**
- 「シンプルさ（YAGNI）」を尊ぶが、金銭が絡む信頼性（監査ログ・不整合防止）については堅牢さを優先する。
- 「ビルドなし（No-Build）」を強力に推進。実装時は `.cursor/rules/no-build-convention.mdc` を厳守すること。
- CSS実装の負担は「完成CSSの自己ホスト」や「デザイントークン + Blade部品化」で減らす（No-Build前提）。Tailwind等のビルド前提FWは原則採用しない。
- No-Build方針と実装の矛盾（package.json/Vite/Tailwind/@vite等の混入）は早期に是正し、方針を単一真実に保つ。

## プロジェクト全体方針
- Laravel 12 / PHP 8.5
- ビルドなし (No Node.js/Vite)
- Blade + CDN系JS (HTMX, Alpine.js) + 生のCSS
- 認証: Laravel Fortify
- 決済: Laravel Cashier (Stripe)
- 冪等性・Webhookを正とする設計
- 予約導線は離脱に強いこと（予約意図/ステート保持、メール認証後の自動復帰、再試行導線）
- 二重予約/二重送信を UI・サーバ両面で防止（disabled 制御 + 冪等性キー）
